# M-249 — API stat no-store: dashboard mai con dati in cache

> Report tecnico (FASE 6a). Esteso: M-249_FULL.md.

- **Organo**: EGI-STAT. Trigger=2 (comportamentale: header HTTP).
- **Sintomo**: dashboard appariva "rotta/ferma" (Indice Produttività, Carico Cognitivo, Mission Chiuse, Commit Pesati a 0; asse settimane fermo al passato) mentre l'API restituiva i dati corretti (W23=168 mission, daily 26 mission/PI 31).
- **Causa**: l'API non inviava header anti-cache → il browser riciclava la risposta GET vecchia anche al reload normale. La dashboard mostrava una foto cache pre-fix.
- **Fix**: `@app.after_request` in api.py → `Cache-Control: no-store, no-cache, must-revalidate, max-age=0` (+ Pragma/Expires) su tutte le risposte `/api/*`. Le statistiche sono dati vivi (auto-refresh dai registry) → mai cacheati. Niente più hard-refresh manuale.
- **Verifica**: header live confermato via curl; test `tests/m_249_nocache_test.py` 1/1; regressione 19/19. Servizio systemd riavviato.
- **Audit** + **DOC-SYNC**: vedi registry.
